# SQE Lab 2: Comparative Quality Analysis
## Topic: Uber vs. inDrive (ISO 25010 Evaluation)

### Overview
This repository contains the laboratory work for **Software Quality Engineering (SQE)**. The objective of this lab is to perform a comparative software quality analysis between two popular ride-hailing applications: **Uber** and **inDrive**, using the **ISO/IEC 25010** quality model.

### Key Metrics (ISO 25010)
The applications were evaluated based on the following 8 quality characteristics:
1.  **Functional Suitability**
2.  **Performance Efficiency**
3.  **Compatibility**
4.  **Usability**
5.  **Reliability**
6.  **Security**
7.  **Maintainability**
8.  **Portability**

### Project Files
* `quality_analysis.py`: Python script to generate the Radar Chart for quality visualization.
* `quality_radar_Uber_inDrive.png`: The generated radar chart illustrating the quality gap between the apps.
* `README.md`: Project documentation.
